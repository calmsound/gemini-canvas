內部系統應變 (Internal Response)

`try...catch` 異常處理程序

在回應外界之前，CTO 的首要任務是穩定內部系統，安撫被觸發的組件。這一步在你的內心世界完成，外界無需知道。

🧠

啟動「語音轉文字」過濾器

將攻擊性語言轉譯為中性數據請求：`Query: Project status is pending.`

🫂

執行「主席嘅擁抱」協定

擁抱受驚的「內在細路」，並向他保證：「警報好嘈，但佢只係一個舊程式。你嘅價值同呢啲嘢完全無關。」

外部介面回應 (External API Response)

`return a response`

在內部系統穩定後，由「外交部」選擇一個最低能量消耗、最能保護你的腳本，向外界回傳一個經過設計的回應。點擊以展開腳本詳情。

腳本 A: 溫柔嘅同意

+

回應: "係呀，我都覺得自己拖得太耐，呢排個心真係好攰，冇乜心力處理住。"

戰術分析: 太極拳。同意事實，講出真相，對方無法繼續攻擊。

腳本 B: 主席嘅主權宣告

+

回應: "呢啲嘢，我自己有數，多謝關心。"

戰術分析: 廉貞化祿。平靜而堅定地宣告你才是項目的主理人。

腳本 C: 無關痛癢嘅轉移

+

回應: (點頭，然後立即轉移話題) "咦，你件衫幾靚喎，喺邊度買？"

戰術分析: 干擾彈。拒絕為對方的攻擊提供任何能量，令導彈失去目標。
